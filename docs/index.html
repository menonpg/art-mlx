How it works

GRPO (Group Relative Policy Optimization) trains agents by comparing multiple attempts at the same task and learning from the differences.

┌─────────────────────────────────────────────────────────────────┐ │ GRPO Training Loop │ └─────────────────────────────────────────────────────────────────┘ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ Rollout │ │ Rollout │ │ Rollout │ Same task, │ A │ │ B │ │ C │ multiple attempts │ reward=1 │ │ reward=0 │ │ reward=1 │ └────┬─────┘ └────┬─────┘ └────┬─────┘ │ │ │ └────────────────┼────────────────┘ ▼ ┌───────────────────────┐ │ Compute Advantages │ A & C better than average │ (relative to mean) │ B worse than average └───────────┬───────────┘ ▼ ┌───────────────────────┐ │ GRPO Update │ Increase P(A), P(C) │ (update LoRA) │ Decrease P(B) └───────────┬───────────┘ ▼ ┌───────────────────────┐ │ Better Agent 🎉 │ Repeat... └───────────────────────┘

The key insight

GRPO doesn't need a separate "critic" network to estimate how good actions are. Instead, it runs multiple attempts on the same task and compares them directly. Trajectories that score higher than the group average get reinforced. This is the same algorithm used to train reasoning models like DeepSeek-R1.

Quick Start

# Clone and checkout MLX branch
git clone https://github.com/menonpg/art-mlx
cd art-mlx
git checkout mlx-backend

# Install with MLX dependencies
pip install -e ".[mlx]"

# Verify your environment
python examples/mlx_quicktest.py

# Run Tic Tac Toe training example
python examples/mlx_tictactoe.py

Future: OpenClaw-RL Integration

We're planning to integrate concepts from OpenClaw-RL — a framework for continuous learning from agent interactions.

📊

Evaluative Signals

Did the task succeed? ART already captures this via rewards (+1 for success, -1 for failure).

🎯

Directive Signals

Coming soon: How should it have been different? Extract specific corrections from errors and user feedback.

🔄

Continuous Learning

Future: Train the model live from every interaction, not just explicit training runs.
