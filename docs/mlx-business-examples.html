Three real-world GRPO examples run end-to-end on Apple Silicon — a natural-language-to-SQL assistant, a customer-support email agent, and a code-review agent. Each fine-tunes a small model on your Mac using LoRA adapters, with no cloud GPU, database, or API keys required.

01What these examples are

Each script follows the same pattern: score the model on a business task, run 8 rounds of GRPO training (reinforcement learning) that nudge trainable LoRA adapters toward higher-scoring outputs, then re-score. They are self-contained — the reward is computed by a built-in heuristic scorer, so they run offline.

02Results summary

All three trained through all 8 steps with real gradient updates. "Best" is the top single-attempt score reached during training; the heuristic scorer runs 0.0–1.0.

Honest caveat. These runs prove the training pipeline works end-to-end on Apple Silicon — not that the model's quality reliably improves. With a tiny 0.5 B model, a keyword-based reward, and only 8 steps, final outputs sometimes degrade (e.g. repetition loops). Real deployments need a larger model, a genuine reward signal (execution results / human feedback), and many more steps.

06Reproduce it

  • Requires Python ≥ 3.12 on Apple Silicon; install with pip install -e ".[mlx]"
  • SQL: python examples/industry/sql_assistant.py
  • Email: python examples/industry/email_support_agent.py
  • Code review: python examples/industry/code_review_agent.py
  • Run long jobs with caffeinate -i python -u ... so sleep can't stall the GPU.
How to make these actually improve. Swap the heuristic score_*() for a real reward (run the SQL and compare rows; measure ticket resolution; track whether devs accept the review), use a bigger base model, and train for far more steps. The plumbing shown here stays the same.
